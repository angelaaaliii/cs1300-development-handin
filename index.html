Development

Utilizing a reactive framework for frontend development to create an aggregator and sorting/filtering functionalities for a small online bookstore.


Background

As part of the research process, we complete a competitive analysis to understand how other organizations are designing their aggregator and sorting/filtering functions.


Competitive Analysis

To start, I selected 3 different websites with favoriting functions:

  • Instagram:

  • Spotify:

  • Amazon:

Then, considering 5 factors:

  1. Ability to sort by date added
  2. Ability to sort items alphabetically
  3. Ability to filter by content type
  4. Filter/sort modal utilized
  5. Filter/sort dropdown utilized
I conducted a competitive analysis with the 5 factors across the 3 chosen websites.

The table below displays the results:

Amazon Spotify Instagram
Sort by date added: no yes yes
Sort alphabetically: no yes no
Filter by content type: yes yes yes
Filter/sort modal: no no yes
Filter/sort dropdown: no yes yes



After conducting the competitive analysis, I found that Instagram's filter/sort modal was least successful because it decreased the user's flow and efficiency by having to navigate away from the favorites list. Along the same reasoning, that also explains why the filter/sort dropdown that Spotify employed was one of the most successful.
Another very successful filtering method involved categorical filtering. Amazon utilized filtering based on type of item and Spotify utilized categorical filtering based on the type of genre. This allows the user to focus on what they are looking for and reduce irrelevant items displayed.
I noticed that sorting alphabetically was only useful for Spotify song titles, which makes sense since these liked songs have official song titles, whereas cart items in Amazon or posts on Instagram don't have titles to alphabetically sort.
Sorting by date also was only useful for Spotify and Instagram because their favorited items accumulate; however for cart items like Amazon, the purpose is to eventually move items from the save for later cart to checkout, so there is no real purpose to sort items saved for later by date added.


Based on these observations and since I am implementing a shopping cart style aggregator, I will not utilize a sort by date functionality. Similar to Spotify songs, since books have proper titles they are referenced by, sorting by titles alphabetically would be successful. I would also like to implement filtering on authors because consumers tend to find their next book based on an author they already like. Lastly, since categorical filtering on content type was successful I would like to implement filtering based on book genre.


Final Site (Bookstore)

The final site developed was a bookstore with a shopping cart for users to add to before checking out.


Bookstore Website Link

  • Key elements include 3 options at the top to sort by titles alphabetically, filter by author, and/or filter by genre.
    • All 3 filters/sorting features can be used together and if any of these dropdown menus are set back to their default state, the filter/sorting will reset.
    • I decided a reset button would be frusturating for the user if they want to control which filters/sorting to reset & having a reset button for each filter/sort functionality would be crowded and overcomplicated for the user
  • Each book item displayed features the title first, then author, price, genre, and an add to cart button.
  • Multiple copies of the same book can be added to the cart.
  • The cart is at the bottom of the page and displays the total price based on the books in the cart.
  • Each title in the cart is next to a remove button that will remove 1 copy of the book from the cart.
  • This way users can control if they want to remove all copies of a book or just one copy of a book from the cart.


Conclusion

To conclude, conducting a competitive analysis really helped when brainstorming functionality to include in my own bookstore website. Looking at different sites with similar features allowed me to get a better idea of what decision choices were successful and unsuccessful.

After examining sorting, filtering, and aggregating on Amazon, Instagram, and Spotify, the final bookstore website was able to combine successful aspects from all 3 sites!
